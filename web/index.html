Permission Demo

This is a demo of using Policy-as-Code tools for authorization control

this assumes you have the Spring app running on localhost on port 8080

johnbr@paclabs.net is an admin user, john.doe@paclabs.net is a regular user, jane.roe@paclabs.net is an editor





devops or finance. johnbr can do anything with both. editors can read finance, create/read/update devops, users can read devops




method=GET for read, method=PUT for create, method=POST for update, method=DELETE for delete
